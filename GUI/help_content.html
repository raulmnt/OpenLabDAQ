OpenLabDAQ

OpenLabDAQ connects laboratory instruments, displays live values, plots recent measurements, and saves synchronized data.

Quick Start

1. Configure the DAQ

  1. Make sure acquisition is stopped.
  2. Click Configuration.
  3. Enable the sensors you want to use.
  4. Select or enter the COM port for each sensor.
  5. Optionally enter a GUI nickname, such as Chamber Temperature.
  6. Select the acquisition period and saving directory.
  7. Click Save.
A nickname changes only the sensor and plot labels shown in the GUI. CSV headers keep the official sensor name.

2. Start Acquisition

Click STOPPED. OpenLabDAQ connects the enabled sensors and then changes the button to RUNNING.

Current values appear on the left and plots update automatically.

3. View and Export History

  • Use the History selector to display the latest 5 minutes, 20 minutes, 2 hours, 6 hours, 1 day, or 3 days.
  • Click the star beside a plot to pin it to the top.
  • Scroll to view additional plots.
  • The vertical plot range adjusts automatically to visible data.
  • Click Export Data to save all sensors in the currently displayed time range to a CSV file.
Export Data works even when normal logging is off, as long as the measurements are still available in History.

4. Start Logging

  1. Start acquisition.
  2. Click LOAD.
  3. Enter optional run information such as experiment, sample, gas, operator, and general notes.
  4. Click Start Logging.

OpenLabDAQ creates two matching files in the configured saving directory: 

2026-07-17_12-58-00.csv
2026-07-17_12-58-00_logbook.txt

The CSV contains sensor measurements. The logbook contains run information and timestamped events.

5. Add an Event

While logging is active, click ADD EVENT.

Enter a short event and an optional comment. OpenLabDAQ automatically adds the current timestamp to the logbook.

Examples include:

  • Gas changed
  • Valve opened
  • Heating started
  • Sample inserted

6. Stop Logging or Acquisition

Click LOGGING to stop saving data and close the logbook.

Click RUNNING to stop acquisition and disconnect all sensors. Stopping acquisition also stops logging.

History and Permanent Files

History is temporary and is stored in computer memory. OpenLabDAQ keeps up to 72 hours during one continuous acquisition session.

History resets when a new DAQ session begins or when OpenLabDAQ closes. Permanent sensor data is created only while the button displays LOGGING, or when you manually use Export Data.

Troubleshooting

Could not open COM port

  • Verify that the instrument is powered.
  • Verify that the USB or serial cable is connected.
  • Check that the correct COM port is selected.
  • Close other software that may already be using the port.

The Configuration button is disabled

Configuration cannot be changed while acquisition is running. Stop acquisition first.

No CSV file is being created

Confirm that acquisition is running and that the logging button displays LOGGING.

ADD EVENT is disabled

Events can be added only while CSV logging and the matching logbook are active.

Export Data reports no data

The selected History range does not contain measurements. Start acquisition and wait for data to appear.

A sensor does not connect

Check its COM port, cable, power, and instrument communication settings. OpenLabDAQ displays the error reported by the sensor driver.

History is not permanent. Use normal logging or Export Data before stopping the program when measurements must be preserved.
