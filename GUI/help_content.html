OpenLabDAQ

OpenLabDAQ is a laboratory data-acquisition program used to connect instruments, display current measurements, plot recent history, and save measurements to CSV files.

Quick Start

1. Configure the DAQ

  1. Make sure acquisition is stopped.
  2. Click Configuration.
  3. Enable the sensors you want to use.
  4. Select or enter the COM port assigned to each sensor.
  5. Select the acquisition period.
  6. Select the directory where CSV files will be saved.
  7. Click Save.
Configuration changes are used the next time acquisition is started. You do not need to close OpenLabDAQ.

2. Start Acquisition

Click STOPPED.

OpenLabDAQ will connect to the enabled sensors. During this process, the sensor panel displays Connecting sensors....

After connection, the button changes to RUNNING. Current sensor values and plots will update automatically.

3. View Plots

Each sensor has its own plot.

  • Use the History selector to display the latest 5 minutes, 20 minutes, 2 hours, 6 hours, 1 day, or 3 days.
  • Click the star beside a plot to pin it to the top.
  • Pinning several plots places them at the top in the order they were pinned.
  • Scroll down to view the remaining plots.
  • Plot vertical ranges adjust automatically using the data visible in the selected time window.

4. Save Data

Data logging is available only while acquisition is running.

  1. Start acquisition.
  2. Click LOAD.
  3. The button changes to LOGGING.
  4. Measurements are written to a CSV file in the configured saving directory.
  5. Click LOGGING again to stop saving data.
Stopping acquisition also stops logging and returns the logging button to LOAD.

5. Stop Acquisition

Click RUNNING.

OpenLabDAQ disconnects the sensors. The last measured values remain visible, and their status indicators become gray.

Understanding History and Logging

Plot history is temporary and is stored in computer memory. OpenLabDAQ keeps up to 72 hours during a continuous acquisition session.

History resets when a new DAQ session begins or when the program is closed.

Permanent data is saved only while the logging button displays LOGGING.

Troubleshooting

Could not open COM port

  • Verify that the instrument is powered.
  • Verify that the USB or serial cable is connected.
  • Check that the correct COM port is selected.
  • Close other software that may already be using the port.
  • Stop acquisition before changing the configuration.

The Configuration button is disabled

Configuration cannot be changed while acquisition is running. Stop acquisition first.

No CSV file is being created

Verify that acquisition is running and that the logging button displays LOGGING.

A sensor does not connect

Check the selected COM port, cable, instrument power, and sensor-specific communication settings.

OpenLabDAQ displays connection and acquisition errors when a sensor driver cannot communicate with its instrument.
